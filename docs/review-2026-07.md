# 版本檢視與優化路線圖（2026-07，v1.2.x）

維護者要求的全面 review：現況強弱項、參考市場產品後的功能取捨、
與接下來一步一步的執行順序。原則不變：零後端、機率一律區間、
樣本不滿 60 天不調參、禁爬 HTML、token 不落地。

## 現況盤點（強項）

- **預測管線**：動態日落窗口（全台/四季）、多模式分歧動態區間、CWA 22 縣市
  交叉驗證、死亡條款/防悲觀/持續性加成，Python canonical + JS parity 測試把關。
- **產品面**：全台 13 點（地區分頁＋定位）、現在的光（餘燼窗口攔截早走）、
  即時影像 lite-facade、判定與行動建議一致性（跳過日不喊出發）、
  三語系（zh/en/es）、首用引導、回報閉環（表單→bot→consensus→隔日加成）。
- **治理**：非商業授權＋NOTICE＋貢獻條款；append-only 日誌紀律；維護者模式。

## 現況弱項（誠實清單）

1. **校準資料幾乎為零**——一切規則常數仍是先驗。這是最大瓶頸，
   而且只能靠時間＋回報累積，工程無法加速（能做的是讓回報更容易，已做）。
2. **即時影像三條連結未經真機驗證**（沙盒開不了 YouTube），且僅北部有。
3. **遮蔽幾何只有台北兩點有實測**——其他點「提前沒入」不會算，
   淡水/西子灣這類有對岸山影的點會高估可見時間。
4. **e2e 未驗證項**：Telegram/ntfy 真實推播（secrets 未設）、ingest bot
   在公開 repo 的完整回報流程、CWA 金鑰在 Actions 的首次真打。
5. **週報/19:15 詢問**只在推播通道存在，app 內沒有對應視圖。

## 路線圖（依序執行；每步都可獨立驗收）

**R1（維護者一次性，擋住其他一切）**
- [ ] 重建 `PUBLIC_REPO_TOKEN`（classic PAT，repo+workflow，建議不過期）→ 跑 sync-public
- [ ] 設 `CWA_API_KEY`；（選）Telegram secrets
- [ ] 真機開三條 YouTube 直播確認畫面 → cams.json `verified: true`

**R2 資料品質（下一輪工程）**
- [ ] 遮蔽幾何補齊：從 DEM（開放高程資料）計算各點西向地平線仰角的
      離線腳本，取代人工目測——一次算 13 點，寫回 viewpoints.json（來源註記）
- [ ] outcome 提醒進 app：19:15 後開 app 自動聚焦「回報」分頁＋角標

**R3 校準準備（等 60 天樣本，先把工具做好）**
- [ ] `python -m sunset calibrate --dry-run`：讀日誌算 Brier/命中率/各情境
      可靠度圖，輸出報告但不改常數（憲章：滿 60 天才調參）
- [ ] 週報加入同樣統計

**R4 體驗深化（隨時可插隊的小步）**
- [ ] 分享卡改圖片（canvas 生成含判定/區間/點位的分享圖）
- [ ] 「地圖上看日落方位」：Google Maps 方位線深連結
- [ ] es/en 的點位名稱補拉丁轉寫（Tamsui、Gaomei…）

**不做清單（重申）**：後端/資料庫、ML、爬 HTML、單點機率、60 天前調參。
